## 3. RAG implementation based on LangChain framework

In the first section, we mentioned that the four steps to build a minimum viable system are data preparation, index construction, retrieval optimization and generation integration. The following will focus on these four aspects to implement a RAG application based on the LangChain framework.

> [Complete code of this section](https://github.com/datawhalechina/all-in-rag/blob/main/code/C1/01_langchain_example.py)

### 3.1 Initialization settings

First perform basic configuration, including importing necessary libraries, loading environment variables, and downloading embedded models.

```python
import os
# os.environ['HF_ENDPOINT'] = 'https://hf-mirror.com'
from dotenv import load_dotenv
from langchain_community.document_loaders import TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_huggingface import HuggingFaceEmbeddings
from langchain_core.vectorstores import InMemoryVectorStore
from langchain_core.prompts import ChatPromptTemplate
from langchain_deepseek import ChatOpenAI

# 加载环境变量
load_dotenv()
```

### 3.2 Data preparation

- **Load original document**: First define the path of the Markdown file, and then use `TextLoader` to load the file as a knowledge source.
    ```python
    markdown_path = "../../data/C1/markdown/easy-rl-chapter1.md"
    loader = TextLoader(markdown_path)
    docs = loader.load()
    ```
- **Chunking**: To facilitate subsequent embedding and retrieval, long documents are divided into smaller, manageable text chunks (chunks). A recursive character segmentation strategy is used here, using its default parameters for chunking. When `RecursiveCharacterTextSplitter()` is initialized without specifying parameters, its default behavior is designed to preserve the semantic structure of the text to the greatest extent possible:
- **Default delimiters with semantic preservation**: Try to recursively split text using a series of preset delimiters `["\n\n" (段落), "\n" (行), " " (空格), "" (字符)]` in sequence. The purpose of this strategy is to keep paragraphs, sentences, and words as intact as possible, since they are usually the most semantically relevant text units, until the text block reaches the target size.
- **Keep delimiters**: By default (`keep_separator=True`), the delimiters themselves are preserved in the split text block.
- **Default block size and overlap**: Uses the default parameters `chunk_size=4000` (block size) and `chunk_overlap=200` (block overlap) defined in its base class `TextSplitter`. These parameters ensure that text blocks conform to predetermined size limits and reduce the loss of contextual information through overlap.
    ```python
    text_splitter = RecursiveCharacterTextSplitter()
    texts = text_splitter.split_documents(docs)
    ```

### 3.3 Index construction

After data preparation is complete, next build the vector index:

- **Initialize Chinese embedded model**: Use `HuggingFaceEmbeddings` to load the Chinese embedded model previously downloaded in the initialization settings. Configure the model to run on CPU with embedding normalization enabled (`normalize_embeddings: True`).
    ```python
    embeddings = HuggingFaceEmbeddings(
        model_name="BAAI/bge-small-zh-v1.5",
        model_kwargs={'device': 'cpu'},
        encode_kwargs={'normalize_embeddings': True}
    )
    ```
- **Build vector storage**: Convert the segmented text blocks (`texts`) into vector representations through the initialized embedding model, and then use `InMemoryVectorStore` to add these vectors and their corresponding original text content to build a vector index in memory.
    ```python
    vectorstore = InMemoryVectorStore(embeddings)
    vectorstore.add_documents(texts)
    ```
Once this process is complete, a knowledge index is built that can be queried.

### 3.4 Query and retrieval

After the index is built, you can query and retrieve user questions:

- **Define User Query**: Set a specific user question string.
    ```python
    question = "文中举了哪些例子？"
    ```
- **Query Vector Store for Related Documents**: Use Vector Store's `similarity_search` method to find the most relevant `k` (`k=3` in the example here) blocks of text in the index based on the user question.
    ```python
    retrieved_docs = vectorstore.similarity_search(question, k=3)
    ```
- **Prepare context**: Combine the retrieved page content of multiple text blocks (`doc.page_content`) into a single string and use double newlines (`"\n\n"`) to separate the blocks to form the final context information (`docs_content`) for reference by the large language model.
    ```python
    docs_content = "\n\n".join(doc.page_content for doc in retrieved_docs)
    ```
> Use `"\n\n"` (double newline) instead of `"\n"` (single newline) to connect different retrieved document chunks, mainly to be able to semantically distinguish these independent text fragments more clearly when passed to a large language model (LLM). Double line breaks usually represent the end of a paragraph and the beginning of a new paragraph. This format helps LLM treat each chunk as an independent source of context, allowing it to better understand and utilize this information to generate responses.

### 3.5 Generate integration

The final step is to combine the retrieved context with the user question and generate an answer using a large language model (LLM):

- **Build prompt word template**: Use `ChatPromptTemplate.from_template` to create a structured prompt template. This template guides LLM in answering the user's question (`question`) based on the context provided (`context`) and clearly indicates how to respond when information is insufficient.
    ```python
    prompt = ChatPromptTemplate.from_template("""请根据下面提供的上下文信息来回答问题。
    请确保你的回答完全基于这些上下文。
    如果上下文中没有足够的信息来回答问题，请直接告知：“抱歉，我无法根据提供的上下文找到相关信息来回答此问题。”
    
    上下文:
    {context}
    
    问题: {question}
    
    回答:"""
                                              )
    ```
- **Configure large language model**: Initialize the `ChatOpenAI` client, configure the model used (`glm-4.7-flash-free`), the temperature parameters for generating answers (`temperature=0.7`), the maximum number of Tokens (`max_tokens=2048`), and the API key (loaded from environment variables) and url.
    ```python
    llm = ChatOpenAI(
        model="glm-4.7-flash-free",
        temperature=0.7,
        max_tokens=2048,
        api_key=os.getenv("DEEPSEEK_API_KEY")
        base_url="https://aihubmix.com/v1"
    )
    ```
- **Calling LLM to generate answers and output**: Format the user question (`question`) and the previously prepared context (`docs_content`) into the prompt template, and then call the `invoke` method of ChatDeepSeek to get the generated answer.
    ```python
    answer = llm.invoke(prompt.format(question=question, context=docs_content))
    print(answer)
    ```

## 4. Low code (based on LlamaIndex)

In terms of RAG, LlamaIndex provides more encapsulated API interfaces, which undoubtedly lowers the threshold for getting started. Here is a simple implementation:

```python
import os
# os.environ['HF_ENDPOINT']='https://hf-mirror.com'
from dotenv import load_dotenv
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader, Settings 
from llama_index.llms.openai_like import OpenAILike
from llama_index.embeddings.huggingface import HuggingFaceEmbedding

load_dotenv()

Settings.llm = OpenAILike(
    model="glm-4.7-flash-free",
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    api_base="https://aihubmix.com/v1",
    is_chat_model=True
)
Settings.embed_model = HuggingFaceEmbedding("BAAI/bge-small-zh-v1.5")

documents = SimpleDirectoryReader(input_files=["../../data/C1/markdown/easy-rl-chapter1.md"]).load_data()

index = VectorStoreIndex.from_documents(documents)

query_engine = index.as_query_engine()

print(query_engine.get_prompts())

print(query_engine.query("文中举了哪些例子?"))
```
